1 What's qLibc?
2 =============
3 
4 qLibc is currently one of the most functionally-complete, publicly-licensed
5 C/C++ libraries. The goal of the qLibc project is to provide a **simple and
6 powerful general purpose C/C++ library** that includes all kinds of containers
7 and general library routines. It provides a ready-made set of common container
8 APIs with a consistent API look.

15 ## API Reference
16 
17 * [qlibc Core API Reference](http://wolkykim.github.io/qlibc/doc/html/files.html)
18  * Containers
19  * Tree Table --- Tree data structure Key/Value table which implements Left-Leaning Red-Black algirothm.
20  * Hash Table --- Hash-based data structure Key/Value pair table.
21  * Static Hash Table --- Fixed size memory structure(array/mmapped/shared) based Key/Value pair table.
22  * List Table --- Linked-list based Key/Value pair table.
23  * List --- Doubly Linked List.
24  * Vector --- implements a growable array of elements.
25  * Queue --- FIFO(First In First Out) implementation.
26  * Stack --- LIFO(Last In First Out) implementation.
27  * General utilities.
28  * String --- string trimmer, modifier, replacer, case converter, pattern detectors, ...
29  * I/O --- non-blcking I/O, stream reader/writer, ...
30  * File --- file locking, file/directory hander, path correctors, ...
31  * IPC, Semaphore Shared-memory
32  * En/decoders --- Url en/decoder, Base64 en/decoder, Hex en/decoder, ...
33  * Hashes --- Murmur hases, FNV hases, MD5 hashes, ...
34  * Time --- time diff, time format converstion, ...
35 
36 * [qLibc Extension API Reference](http://wolkykim.github.io/qlibc/doc/html/files.html)
37  * Apache-style Configuration File Parser.
38  * INI-style Configuration File Parser.
39  * HTTP client.
40  * Rotating File Logger.
41  * Database(MySQL) interface.
42  * [Token-Bucket](http://en.wikipedia.org/wiki/Token_bucket)

44 ## Consistent API Look
45 
46 All container APIs have a consistent look and feel. It basically provides
47 a creator function which usually returns a pointer to a container structure.
48 Also, **all functions related to the container can be accessed through function
49 pointers inside of the container**.
50 
51 So, regardless of which container you use, you can simply put elements into
52 a list with `pContainer->put(pContainer, ...)`. It looks like C++ classes,
53 but it's a pure C implementation. Of course, it works with both C and C++ code**.
54 
55 We use this concept as much as possible because it is a simplified way of
56 thinking and helps to improve readability. Consequently, it helps people to
57 write bug-free code more easily.
58 
59 The examples below illustrates how qlibc looks like at a glance.
60 
61 ~~~{.c}
62  // create a hash-table.
63  qhashtbl_t *tbl = qhashtbl(0, QHASHTBL_OPT_THREADSAFE);
64 
65  // add an element which key name is "score".
66  int x = 12345;
67  tbl->put(tbl, "score", &x, sizeof(int));
68 
69  // get the value of the element.
70  int *px = tbl->get(tbl, "score");
71  if(px != NULL) {
72  printf("%d\n", *px);
73  free(px);
74  }
75 
76  // release table
77  tbl->free(tbl);
78 ~~~
79 
80 Here is an identical implementation with a Linked-List-Table container.
81 You may notice that there aren't any code changes at all, except for 1 line
82 in the table creation. This is why qLibc encapsulates corresponding function
83 pointers inside of the container object.
84 
85 ~~~{.c}
86  // create a linked-list-table. THE ONLY LINE YOU NEED TO CHANGE.
87  qlisttbl_t *tbl = qlisttbl(QLISTTBL_OPT_THREADSAFE);
88 
89  // add an element which key name is "score".
90  int x = 12345;
91  tbl->put(tbl, "score", &x, sizeof(int));
92 
93  // get the value of the element.
94  int *px = tbl->get(tbl, "score");
95  if(px != NULL) {
96  printf("%d\n", *px);
97  free(px);
98  }
99 
100  // release table
101  tbl->free(tbl);
102 ~~~
